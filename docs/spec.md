# レンズラインナップ表 仕様書

## 1. 概要

ミラーレスカメラ用レンズのラインナップを焦点距離軸で視覚化する静的Webページ。
GitHub Pages でホスティングする。**React + TypeScript（Vite）** で構成する。

### 1.1 マルチマウント対応

以下の3マウントに対応し、マウントごとに個別のパスで表示する。

| マウント ID | 表示名 | パス | 主なメーカー |
|---|---|---|---|
| `z` | Zマウント | `/lens-list/z` | ニコン・タムロン・シグマ |
| `e` | Eマウント | `/lens-list/e` | ソニー・タムロン・シグマ |
| `rf` | RFマウント | `/lens-list/rf` | キヤノン・シグマ・タムロン |

- ルートパス（`/lens-list/`）およびマウントに一致しないパスへのアクセスは、クライアントサイドで `/lens-list/z` に置き換える（`history.replaceState`）。
- マウント定義は `src/lib/mounts.ts` で管理する。
- `document.title` は `lens-list | <マウント表示名>` に設定する。
- E マウント（128本）・RF マウント（53本）は調査済みデータを収録済み。価格は実勢価格の概算、レンタル・公式URLは大部分が未設定。発売日は精度が低いものが含まれるため、随時精査して更新する。
- 焦点距離マーカーの範囲はマウントごとに異なる（E: 〜800mm、RF: 〜1200mm）。10mm 台のレンズがあるマウントは原点マーカーの `spacingToNext` を広げて左余白を確保する。

### 1.2 マウント別パスの配信（GitHub Pages）

GitHub Pages は SPA のパスフォールバックを持たないため、ビルド後に `scripts/postbuild.mjs` が `dist/index.html` を `dist/z/`・`dist/e/`・`dist/rf/` に複製する（`npm run build` に組み込み済み）。

## 2. 用語定義

| 用語 | 定義 |
|---|---|
| **レンズ (Lens)** | 個々のレンズ商品。最小単位。 |
| **行 (Row)** | セクション内で同じ縦位置に並ぶレンズの集合。1本のみでも行を構成する。 |
| **セクション (Section)** | 行を束ねる大区分。例：単焦点レンズ / 薄型単焦点レンズ / ズームレンズ。 |
| **表示属性 (Display Attribute)** | 重さ・価格・レンタル有無など、表中のレンズ名の下に表示できる属性。表の上のチェックボックスで表示／非表示を切り替える。 |
| **フィルター (Filter)** | 焦点距離・最大絞り・価格・レンタル可否などの条件でレンズを絞り込む機能。 |
| **マウント (Mount)** | レンズマウント（Z / E / RF）。マウントごとに独立したパス・設定・データを持つ。 |
| **設定ファイル** | `config.json`。表示に関するすべての定数を管理する。**マウントごとに1つ**（`public/data/<mountId>/config.json`）。 |
| **レンズデータファイル** | `lenses.json`。レンズの属性・レイアウト情報を管理する。**マウントごとに1つ**（`public/data/<mountId>/lenses.json`）。 |

### 4.1 全体レイアウト

### 4.2 焦点距離ヘッダー

- 表の最上部に FX 換算値を第1行、DX 換算値を第2行（参考値・やや小さく薄く表示）として常時表示する。
- 焦点距離の基準は常に FX 換算値とし、フィルター・配置ともに FX 換算値を使用する。DX 行は対応する FX 値の参考情報として表示するにとどめる。
- 各マーカー間の幅は `config.json` の `focalLengthMarkers` の `spacingToNext` で定義する。
- 左隅のフォーマットラベルは `config.json` の `formatLabels` で定義する（Z: `FX`/`DX`、E/RF: `フルサイズ`/`APS-C`）。ラベルが4文字以上の場合はフォントを縮小して表示する。
- DX 行の換算値（`dxMm`）はマウントのクロップファクター（`cropFactor`。Z/E: 1.5、RF: 1.6）に合わせて config で定義する。

### 4.3 セクション

- セクション数は可変（`lenses.json` に定義した数だけ生成する）。
- セクションラベルは表の左端に縦書きで表示する。
- セクションの背景色は `config.json` で指定する。
- **レスポンシブ**：PC（768px以上）ではセクションラベルおよびヘッダー左端コーナーが `position: sticky; left: 0` で左端に固定表示される。スマホ（767px以下）では sticky を解除し、通常フローで表示する。

### 4.4 行

- 行数は可変（`lenses.json` に定義した数だけ生成する）。
- 同一行内のレンズは同じ縦位置（上端揃え）に配置する。
- 行の高さはすべての行で共通だが、表示属性チェックボックスの状態に応じて動的に変化する。
- 行の高さ＝`rowBaseHeightPx`＋（現在オンの表示属性数 × `attributeRowHeightPx`）
- チェックボックスの切り替え時に全行の高さを一括で再計算・再描画する。

### 4.5 レンズのホバー・クリック操作

- マウスカーソルをレンズ（ドットまたはバー）に乗せると、レンズ全体が上方向に 4px 浮き上がるアニメーションを表示する（`transition: transform 0.15s ease`）。どのレンズが選択対象かを視覚的に示す。
- **クリック（PCマウス）またはタップ（スマホ）**で、そのレンズの詳細情報モーダルを表示する。
- レンズ要素は `role="button"` / `tabIndex={0}` を持ち、キーボードの Tab でフォーカス移動、Enter または Space キーでも詳細モーダルを開ける。フォーカス時は `:focus-visible` でアウトラインを表示する。

### 4.6 レンズ詳細モーダル

- 設定モーダルと同じ iPhoneスタイルのボトムシートモーダルを使用する（下から上へスライドイン）。
- モーダルのヘッダーにレンズ名（NEWバッジ付き）と ✕ 閉じるボタンを表示する。
- 本文には以下の情報を2セクションで表示する：
  - **固定フィールド（常時表示）**：メーカー、フォーマット（FX/DX）、焦点距離、最大絞り
  - **`config.displayAttributes` の全属性**：`show_rental=1` の場合のみ「レンタル」を含む（`effectiveConfig` による制御）
- ✕ ボタン・オーバーレイクリック・Esc キーでモーダルを閉じる。
- モーダル表示中はボディのスクロールをロックする。
- レンタル価格（`rentalAvailable`）は URL パラメーター `show_rental=1` がある場合のみ表示する（設定モーダルのレンタル表示制御と連動）。
- 本文の最下部に「Google検索」ボタン（塗りつぶしカプセル）を表示する。押下するとレンズ名をクエリとして Google 検索を新しいタブで開く。
- `officialUrl` が設定されているレンズには、「Google検索」の下に「公式サイト」ボタン（白背景・縁取りカプセル）を表示する。押下するとメーカー公式製品ページを新しいタブで開く。現在はニコンレンズのみに設定されている。

### 4.7 レンズの描画

#### 単焦点レンズ

- **図形**：塗りつぶし円（●）
- 円の中心が該当する焦点距離のマーカー位置に来るよう配置する。
- DX フォーマット専用レンズは**表記上の焦点距離（実焦点距離）**の位置に配置する。FX 換算はしない。`lenses.json` の `focalLengthFxMm` には `focalLengthMm` と同じ値を設定する。
- 円の直径は `config.json` の `primeDotDiameterPx` で指定する。

#### ズームレンズ

- **図形**：角丸付き長方形（横バー）
- 左端が最短焦点距離のマーカー位置に、右端が最長焦点距離のマーカー位置に一致するよう配置する。DX 専用ズームレンズも同様に**表記上の焦点距離（実焦点距離）**で配置する（`focalLengthMinFxMm`・`focalLengthMaxFxMm` は `Min/MaxMm` と同値）。
- 高さ・角丸半径は `config.json` で指定する。

#### レンズ名表示

- 図形の直下にレンズ名を表示する。
- 有効な表示属性があれば、レンズ名のさらに下に縦に並べる。

#### NEWバッジ

- 発売日（`releaseDate`）がページ表示時点から **6ヶ月以内** のレンズには、レンズ名の左側に赤い「NEW」バッジを表示する。
- バッジの表示・非表示は設定モーダルの「新製品(半年以内)」トグルで切り替える（デフォルトON）。
- `releaseDate` が未設定のレンズはバッジを表示しない。

### 4.8 トップバー

- 画面最上部に固定表示する。
- 左側：**マウント選択ボタン**（例:「▽ Zマウント」。現在表示中のマウント名を表示）。
  - クリックするとドロップダウンメニューが開き、「Zマウント」「Eマウント」「RFマウント」を選択できる。
  - 各項目は対応するマウントのパス（`/lens-list/z` 等）へのリンク（ページ遷移）。現在のマウントはハイライト表示する。
  - メニューは外側クリック・Esc キーで閉じる。
- 右側：「⚙ 設定」「📷 画像保存」の2つのカプセルボタンを横並びで配置する。各ボタンは独立したカプセル形状（`border-radius: 999px`）。

### 4.9 設定モーダル

- 「設定」ボタン押下で iPhoneスタイルのボトムシートモーダルが下からスライドアップして表示される。
- モーダル内には「表示属性」セクションと「フィルター」セクションを縦に並べる（アコーディオンなし・常時展開）。
- 右上の ✕ ボタン、またはオーバーレイ部分をクリック、Escキーでモーダルを閉じる。
- モーダル表示中はボディのスクロールをロックする。

### 4.10 テーブルメタ情報バー

- 表の直上（スクロール外）に1行のバーを表示する。
- 左側：免責テキスト（「※ 掲載情報は参考目的のみです。価格・仕様は変更される場合があります。最新情報はメーカー公式サイトをご確認ください。」）。コンテナ幅が不足する場合は marquee アニメーションで横スクロール表示する。
- 右側：最終更新日（`lenses.json` の `lastUpdated` が存在する場合のみ表示）。

## 5. 表示属性チェックボックス

- 設定モーダル内の「表示属性」セクションに横並びで配置する（折り返し対応）。
- チェックを入れた属性のみ、表中のレンズ名の下に表示する。

### 5.1 新製品トグル（固定）

「新製品(半年以内)」は先頭に固定配置される特別なトグルで、NEWバッジの表示を制御する。

- **デフォルト：オン**
- オンのとき、発売から6ヶ月以内のレンズにNEWバッジを表示する。
- `config.json` ではなくコード側（`useActiveAttributes` フック）で管理する。

### 5.2 表示属性一覧

初期状態はすべてオフ。属性の種類・ラベルは `config.json` で定義し、コードを変更せずに追加・削除できる。

| 属性キー | 表示ラベル | 備考 |
|---|---|---|
| `weightG` | 重さ | 重さ (g) |
| `priceJpy` | 価格 | 価格 (円) |
| `releaseDate` | 発売日 | `YYYY-MM-DD` 形式で保存。`YYYY年M月D日` 形式で表示。 |
| `campaignCashbackJpy` | キャンペーン（2026夏）| キャッシュバック金額。`CB: XX,XXX円` 形式。対象外は `×` |
| `rentalAvailable` | レンタル | レンタル料金 (円/泊) |

#### キャンペーン属性について

- キャンペーン属性は現在 **Zマウントのみ**で使用する（E/RF の config には含めない）。
- `campaignCashbackJpy` フィールドはキャンペーンのキャッシュバック金額を管理する。
- 対象レンズ（ニコン）：金額（例: `20000`）
- 対象外レンズ（ニコン非対象・他メーカー）：`false`
- 表示フォーマット：`CB: XX,XXX円`（対象）／ `×`（非対象）
- キャンペーン切替時は `config.json` のラベル（例: `キャンペーン（2026春）` → `キャンペーン（2026秋）`）と `lenses.json` の各レンズの `campaignCashbackJpy` 値を手動で更新する。

## 6. レンズデータの行配置ルール

### 6.1 行のグループ化方針

- セクション内の行は**上からおおむね f 値の明るい順**に並べる。
- 同一行に配置するレンズは**最大絞り（f値）ができるだけ同じ**になるよう揃える。
- **意味のあるレンズ群**（大三元・小三元・F11超望遠ペア・マクロ群・F1.4シリーズなど）は可能な限り同一行にまとめる。ただし焦点距離レンジが物理的に重なる場合（例: RF 15-35mm と 24-70mm）はバーが重なるため隣接行に分ける。
- 同一行内で焦点距離が重複する、または近接してレンズ名が重なる可能性がある場合は別行に分ける。
- マクロレンズ（MC）は単焦点レンズとして扱い、単焦点セクションに配置する。
- Special Edition（SE）版は通常版が存在する場合は除外する。
- 後継機（II など）が存在する場合は旧モデルを除外し後継機のみ掲載する。

### 6.2 DX レンズの行配置

- DX レンズは各セクションの**下部の行**にまとめて配置する。
- FX レンズと DX レンズが同一行に混在しないことが望ましい。
- パワーズーム（PZ）レンズは単独行とする。

### 6.3 現在のセクション構成

全マウント共通で以下のセクション構成とする。純正レンズは種別（単焦点／薄型単焦点／ズーム）で分け、**サードパーティ（タムロン・シグマ）はメーカー別セクション**に配置する。

| セクション ID | ラベル | 内容 |
|---|---|---|
| `prime` | 単焦点レンズ | 純正のフルサイズ単焦点全般（上位ライン・マクロ含む）、APS-C 単焦点（最下行） |
| `compact-prime` | 薄型単焦点レンズ | 純正の小型・軽量・パンケーキ系（Z 26・28・40mm、RF 16・28mm 等） |
| `zoom` | ズームレンズ | 純正のフルサイズズーム全般、APS-C ズーム（最下行・PZ は単独行） |
| `tamron` | タムロン | タムロン製レンズ全般。**単焦点・ズーム混在可** |
| `sigma` | シグマ | シグマ製レンズ全般。**単焦点・ズーム混在可** |

## 7. フィルター機能

フィルター項目は設定モーダル内の「フィルター」セクションに常時展開して表示する（アコーディオンなし）。
適用中のフィルターは、トップバーとテーブルメタ情報バーの間に**フィルターチップバー**として表示する。
チップバーは横スクロール可能な1行で、適用中の各フィルターをテキストチップとして並べる。チップの表示形式は以下の通り。

| フィルター | チップ表示例 |
|---|---|
| 焦点距離 | `焦点距離: 24〜70mm` |
| 最大絞り | `最大絞り: f/2.8以下` |
| 価格 | `価格: 〜300,000円` |
| レンタル可否 | `レンタル可` |
| フォーマット | `フォーマット: FX対応` / `フォーマット: DX専用` |
| 手ぶれ補正 | `手ぶれ補正` |
| メーカー | `メーカー: タムロン` |
| Sライン | `Sライン` |

フィルターが1件も適用されていない場合はチップバーを非表示にする。
各チップには×ボタンを設け、クリックするとそのフィルターを単独で解除できる。
フィルターに一致しないレンズは非表示にする。
その結果、表示すべきレンズが0件になった行も非表示にする。
さらに、セクション内のすべての行が非表示になった場合はセクションごと非表示にする。

| フィルター名 | 種別 | 対象属性 |
|---|---|---|
| 焦点距離 | 範囲スライダー（min / max） | `focalLengthFxMm`（単焦点）または `focalLengthMinFxMm` 〜 `focalLengthMaxFxMm`（ズーム）。DX レンズも表記上の焦点距離（実焦点距離）で評価する。 |
| 最大絞り | セレクトボックス（以下） | `maxAperture` |
| 価格 | 範囲スライダー（min / max） | `priceJpy`。min は 0 固定、max はデータから動的に計算。 |
| レンタル可否 | トグル | `rentalAvailable` |
| フォーマット | セレクトボックス（すべて / FX / DX） | `format` |
| 手ぶれ補正 | トグル | `imageStabilization` |
| メーカー | セレクトボックス（すべて / マウント別の選択肢） | `manufacturer` |
| Sライン（ブランドライン） | トグル | `sLine`。純正上位ラインのフラグとしてマウント共通のキー `sLine` を使用し、ラベルはマウント別 config で定義する（Z: Sライン、E: G Master、RF: Lレンズ）。フィールド未設定のレンズは `false` 相当。 |

フィルターの種類・パラメーターは `config.json` で定義する（マウントごとに選択肢・ラベルが異なる）。
「フォーマット」フィルターの選択肢表示は `formatLabels` に従う（例: E マウントでは「フルサイズ対応」「APS-C専用」）。

## 8. 画像保存機能

- トップバー右側の「📷 画像保存」カプセルボタンから操作する。
- クリックすると `html2canvas` を使用してテーブル全体を PNG 画像として保存する（ファイル名：`lens-list-<マウントID>.png`。例: `lens-list-z.png`）。
- 保存中はボタンを「保存中…」テキストに変更し、無効化する。
- 画像キャプチャ前にスクロール位置をリセットし、キャプチャ後に元の位置へ復元する。

## 9. config.json 仕様

マウントごとに `public/data/<mountId>/config.json` を配置する。マウント間で異なる主な項目：

| 項目 | Z | E | RF |
|---|---|---|---|
| `formatLabels` | `FX` / `DX` | `フルサイズ` / `APS-C` | `フルサイズ` / `APS-C` |
| `cropFactor` | 1.5 | 1.5 | 1.6 |
| メーカーフィルター options | ニコン・タムロン・シグマ | ソニー・タムロン・シグマ | キヤノン・シグマ・タムロン |
| `sLine` フィルターのラベル | Sライン | G Master | Lレンズ |
| キャンペーン表示属性 | あり | なし | なし |

- `select` 型フィルター：初期値は `"all"`（全件表示）。
- `toggle` 型フィルター：初期値は `false`（絞り込みなし）。

## 11. 非機能要件

| 項目 | 要件 |
|---|---|
| ホスティング | GitHub Pages（静的配信）。マウント別パスは `scripts/postbuild.mjs` による `index.html` 複製で対応 |
| 技術スタック | React 19 + TypeScript 5 + Vite 7 |
| 外部ライブラリ | `html2canvas`（画像保存のみ） |
| ブラウザ対応 | Chrome / Firefox / Safari 最新版 |
| レスポンシブ | 横スクロール対応（表は最小幅を維持し、画面幅が不足する場合は横スクロール） |
| アクセシビリティ | チェックボックス・フィルターには `<label>` を付与する。レンズ要素は `role="button"` / `tabIndex` / キーボード操作（Enter・Space）に対応する |
| Sticky 固定 | 焦点距離ヘッダー行（FX / DX）は縦スクロール時に画面上部へ固定（`position: sticky; top`）。セクションラベル列は横スクロール時に画面左端へ固定（`position: sticky; left`）。両者が交差するコーナーセルは両方向に対して固定する。 |
